# Design — Психология масштаба личности

Визуальный контракт учебного проекта AgentOS: продающий лендинг и Telegram-бот с админскими функциями. Стиль построен на брендбуке «Психология масштаба личности» (ПМЛ): зрелый, сдержанный, архитектурный, премиальный.

Этот документ — источник правды по визуальному языку проекта. Любая вёрстка, новый экран, новая секция, новый компонент админки должны сверяться с этим файлом.

## Design principles

- **Воздух важнее плотности.** Пустое пространство — это часть композиции, а не «недоделанный экран». Не заполняем «лишь бы не было пусто».
- **Иерархия и тишина.** На экране всегда видно, что главное. Второстепенное визуально отступает: меньше контраст, меньше вес, больше отступы вокруг главного.
- **Центральная ось.** Длинные смысловые блоки выстраиваются по центральной вертикальной оси. Декоративные смещения не используются ради «динамики».
- **Деликатные разделители.** Между смысловыми блоками — тонкая линия, ромб или точка. Не используем жирные плашки и контрастные блоки-разделители.
- **Бронза дозированно.** Bronze и Deep Bronze — это смысловой и статусный акцент: CTA, ключевая цифра, иконка статуса, бренд-знак. Никогда не используются как «общий фон секции» или для украшения.
- **Tone of voice — глубоко, ясно, спокойно, по сути.** Тексты на экране, microcopy кнопок и подсказки в админке избегают инфобиз-клише, мотивационного шума, эзотерики, агрессивного пафоса и высокомерия.
- **Зрелость, а не «продающее давление».** Шрифт, цвет, отступ говорят громче, чем восклицательные знаки. Никогда не используем «КУПИ СЕЙЧАС», капс на CTA, мигающие элементы, таймеры дефицита.

## Color tokens

Основная тема — `light`. На `Ivory Light` строится весь лендинг и админка по умолчанию. Допустимы отдельные секции на `Digital Night` — например, hero, манифест, цитата основателя, экран входа в админку.

### Status tokens

Статусные цвета построены вокруг бронзы и графита, без кислотных значений. Используются точечно в админке: индикаторы заявок, статусы рассылок, валидация форм.

| Token | HEX | Назначение |
| --- | --- | --- |
| `--status-success` | `#5C7A4A` | Успех: заявка принята, рассылка ушла |
| `--status-warning` | `--color-bronze` | Внимание: требуется действие администратора |
| `--status-danger` | `#8C3A2E` | Ошибка: отказ, провал валидации |
| `--status-info` | `--color-graphite` | Нейтральный системный статус |

### Контраст

- Основной текст `--text-primary` на `--bg-page` обеспечивает контраст не ниже WCAG AA для обычного текста.
- CTA `--accent` на `--bg-page` используется только для крупного текста и кнопок размера M и выше. Для мелких ссылок в тексте используем `--text-accent` (`Deep Bronze`).
- На digital-секциях основной текст — `--text-primary-dark` на `--bg-page-dark`, контраст не ниже WCAG AA.

## Typography

Три гарнитуры с разной ролью. Никогда не используются взаимозаменяемо.

### Families

| Token | Шрифт | Роль |
| --- | --- | --- |
| `--font-display` | Literata | Заголовки, титульные акценты, брендовые фразы, цитаты |
| `--font-body` | Lora | Основной текст, длинное чтение, методические блоки, описания |
| `--font-ui` | Manrope | Интерфейс: кнопки, поля ввода, подписи, метки, навигация, таблицы админки |

Fallback-стек: `"Literata", "Lora", Georgia, serif` для serif-гарнитур; `"Manrope", "Inter", system-ui, sans-serif` для UI.

### Правила

- Заголовки `Literata` — только в `--text-primary` или `--text-primary-dark`. Не используем бронзу для заголовков целиком: бронза — для одного выделенного слова или строки внутри заголовка.
- Длинный текст набираем `Lora` шириной не более 68 символов в строке.
- `Manrope` для всего интерфейсного: кнопки, инпуты, бэйджи, таблицы, навигация в админке, microcopy.
- Никогда не используем все три гарнитуры в одном смысловом блоке без необходимости. Базовая пара секции: `Literata` заголовок + `Lora` текст; в админке: `Manrope` интерфейс + `Lora` для описаний и пояснений.
- Не используем `italic` для эмоционального усиления. `italic` допустим только для цитат и названий внешних работ.

## Spacing and layout

### Spacing scale (8 px база)

| Token | px | Применение |
| --- | --- | --- |
| `--space-1` | 4 | Микро-зазоры внутри бэйджа, между иконкой и текстом |
| `--space-2` | 8 | Внутренние отступы малых элементов, gap в таблицах |
| `--space-3` | 12 | Поля ввода, мелкие карточки |
| `--space-4` | 16 | Базовый внутренний padding карточек |
| `--space-5` | 24 | Стандартный gap между элементами одного блока |
| `--space-6` | 32 | Отступы внутри секций, gap между карточками |
| `--space-8` | 48 | Отступы вокруг заголовков секций |
| `--space-10` | 64 | Вертикальный ритм между подсекциями |
| `--space-12` | 96 | Стандартный вертикальный отступ между секциями лендинга |
| `--space-16` | 128 | Усиленный отступ вокруг ключевых секций: hero, манифест, CTA |

### Container & grid

- Максимальная ширина основного контейнера лендинга — `1200px`.
- Контейнер длинного чтения (манифест, методика) — `720px` для оптимальной читаемости `Lora`.
- Админка использует ширину `1440px` максимум, с боковой навигацией `260px` и рабочей зоной.
- Базовая сетка: 12 колонок, gutter `--space-6` (32 px) на desktop, `--space-4` (16 px) на мобиле.
- Внешние поля контейнера: desktop `--space-12`, tablet `--space-8`, mobile `--space-5`.

### Радиусы

| Token | px | Применение |
| --- | --- | --- |
| `--radius-sharp` | 0 | Архитектурные блоки, hero-плашки, фотографии |
| `--radius-s` | 4 | Поля ввода, бэйджи, мелкие чипы |
| `--radius-m` | 8 | Карточки, таблицы, модальные окна |
| `--radius-l` | 16 | Только крупные premium-карточки (тариф, ключевое предложение) |

Скруглений больше 16 px не используем. Никаких pill-кнопок (`border-radius: 999px`) — это противоречит архитектурному характеру стиля.

### Тени

Тени почти не используем. Глубина создаётся бордером и фоном.

| Token | Значение | Применение |
| --- | --- | --- |
| `--shadow-none` | `none` | По умолчанию |
| `--shadow-soft` | `0 1px 2px rgba(35, 42, 49, 0.06)` | Hover карточки, выпадающие меню |
| `--shadow-elevated` | `0 12px 32px rgba(35, 42, 49, 0.12)` | Модальные окна, тосты |

## Components

### Buttons

Три варианта, никаких декоративных градиентов и обводок-неонов.

- **Primary** — фон `--accent`, текст `--color-ivory-light`, шрифт `Manrope 500`, размер `--text-ui-m`, padding `--space-3` по вертикали и `--space-5` по горизонтали, радиус `--radius-s`. Hover — фон `--accent-hover`. Используется для главного действия экрана: «Оставить заявку», «Войти в админку», «Опубликовать рассылку».
- **Secondary** — фон `transparent`, текст `--text-primary`, бордер `1px solid --border-strong`, остальное как у primary. Hover — фон `--bg-surface`. Используется для второстепенных действий рядом с primary.
- **Ghost** — фон `transparent`, текст `--text-accent`, без бордера. Hover — подчёркивание `1px` `--accent-hover`. Используется для текстовых действий: «Подробнее», «Отменить».

На экране одновременно — не больше одного `Primary` действия в зоне видимости.

### Inputs

- Фон `--bg-surface`, бордер `1px solid --border-hairline`, радиус `--radius-s`, шрифт `Manrope 500 / --text-ui-m`.
- В фокусе: бордер `--border-strong`, без свечения. Не используем box-shadow в фокусе.
- Лейбл сверху, не плейсхолдером. Плейсхолдер — `--text-muted`.
- Ошибка: бордер `--status-danger`, подпись под полем `--text-ui-s` цветом `--status-danger`.

### Cards

- Базовая карточка: фон `--bg-surface`, бордер `1px solid --border-hairline`, радиус `--radius-m`, padding `--space-6`, gap внутри `--space-4`.
- Premium-карточка (тариф, оффер): радиус `--radius-l`, бордер `1px solid --color-bronze`, заголовок с одним словом-акцентом в `--color-bronze-deep`.
- Карточка статуса в админке: радиус `--radius-s`, левая полоса `3px solid` цветом статусного токена.

### Navigation

- Шапка лендинга: фон `--bg-page`, нижний бордер `--border-hairline`, высота `72px`, лого слева, навигация по центру или справа, CTA `Primary` справа.
- Боковая навигация админки: фон `--bg-page`, правый бордер `--border-hairline`, пункты `Manrope 500 / --text-ui-m`, активный пункт — текст `--color-graphite`, левая полоса `3px solid --color-bronze`.
- Никаких выпадающих мега-меню. Максимум — один уровень вложенности.

### Tables (admin)

- Заголовок таблицы: `Manrope 600 / --text-ui-s`, цвет `--text-secondary`, uppercase, tracking `0.04em`.
- Строка: `Lora 400 / --text-body-s` или `Manrope 500 / --text-ui-m` в зависимости от типа данных. Высота строки — `--space-8` (48px).
- Разделитель строк — `--border-hairline`, без zebra-полос.
- Статусные ячейки — бэйдж с фоном `rgba` соответствующего статусного цвета `0.12` и текстом полным цветом статуса.

### Section dividers

- Тонкая линия: `1px solid --border-hairline`, ширина не более `--space-16` (128px), по центру.
- Ромб-разделитель: квадрат `8x8` цвета `--color-bronze`, повёрнут на 45°, центрирован, с воздухом `--space-8` сверху и снизу.
- Точка-разделитель: круг `4x4` цвета `--color-graphite` с `opacity 0.4`, по центру.

### Bot admin specifics

- Карточка заявки: левая статусная полоса, заголовок `Manrope 600 / --text-heading-s`, мета строка `--text-ui-s` цветом `--text-muted`, тело `Lora 400 / --text-body-s`.
- Карточка рассылки: превью текста сообщения шрифтом `Manrope 500 / --text-ui-m` на фоне `--bg-elevated`, бордер `--border-hairline`, ширина — как у Telegram-сообщения (максимум 420px).
- Кнопки действий с заявкой: `Primary` «Принять», `Secondary` «Отложить», `Ghost` «Отклонить». Никогда не используем `--status-danger` для текста или фона кнопки «Отклонить» — статус показывается отдельной иконкой/бэйджем.

## Responsive rules

### Breakpoints

| Token | Значение | Назначение |
| --- | --- | --- |
| `--bp-sm` | `640px` | Мобильные устройства, портрет |
| `--bp-md` | `768px` | Планшет, портрет |
| `--bp-lg` | `1024px` | Планшет landscape, маленький ноутбук |
| `--bp-xl` | `1280px` | Десктоп |
| `--bp-2xl` | `1536px` | Большой десктоп |

### Поведение

- **Mobile (< 640):** одноколоночная вёрстка, внешние поля `--space-5`, hero-заголовок `--text-display-l` уменьшается до `40 / 48`, body — `--text-body-m` (`17 / 28`).
- **Tablet (640–1023):** двухколоночные блоки допустимы для карточек, навигация в шапке коллапсируется в меню-кнопку при ширине меньше `768px`.
- **Desktop (≥ 1024):** полная сетка 12 колонок, fixed-шапка лендинга, боковая навигация админки видна постоянно.
- Вертикальный ритм между секциями: mobile `--space-10`, tablet `--space-12`, desktop `--space-16`.
- Никогда не уменьшаем размер CTA-кнопки ниже высоты `44px` — это нижняя граница тач-таргета.
- Изображения и фото — `object-fit: cover`, `--radius-sharp` или `--radius-m`. Никаких круглых аватаров крупного размера: круглые аватары допустимы только до `40x40`.

## Anti-patterns

Этого в проекте быть не должно — ни на лендинге, ни в админке, ни в Telegram-сообщениях бота.

### Цвет

- Кислотные, неоновые, насыщенно-яркие цвета вне палитры: чистый красный, неоновый зелёный, электрический синий, фуксия.
- Градиенты на больших площадях, особенно радужные и «закат».
- Использование `--color-bronze` как фон большой секции.
- Использование статусных цветов как декоративных.
- Текст `--color-bronze` на `--bg-surface` — не проходит по контрасту.

### Композиция и стиль

- Перегруженные экраны: больше одного `Primary` CTA в зоне видимости, плотные сетки без воздуха.
- Декоративные иконки без смыслового значения, «рамочки ради рамочек», лишние разделители.
- Скругления больше 16 px и pill-кнопки.
- Глубокие/цветные тени, glow, neon-эффекты, glassmorphism.
- Анимации длиннее 250 ms, параллакс на скролле, мигание, автопроигрывание видео со звуком.
- Стоковые фото с улыбающимися людьми в офисе, рукопожатия, графики со стрелками вверх.

### Типографика

- Капс на заголовках длиннее 3 слов.
- Капс на CTA-кнопках.
- Восклицательные знаки в кнопках и заголовках секций.
- Смешивание `Literata` и `Lora` в одном заголовке.
- Использование `Manrope` для длинных текстов и описаний.
- Курсив для «эмоционального усиления».
- Тексты шире 68 символов в строке для длинного чтения.

### Tone of voice

- Инфобиз-клише: «прорыв», «трансформация», «секретная методика», «успех за 7 дней».
- Мотивационный шум: «ты сможешь», «поверь в себя», «начни прямо сейчас».
- Эзотерическая размытость: «энергии», «вибрации», «активация потенциала» без конкретики.
- Агрессивный пафос: «только для избранных», «элита», «закрытый клуб» в качестве давления.
- Высокомерный тон: «ты не понимаешь», «обычные люди не способны».
- Дефицит и срочность: таймеры, «осталось 3 места», «цена вырастет завтра».
- Эмодзи в заголовках и CTA. В тексте лендинга — не используем. В сообщениях бота допустимы только функциональные символы статуса.
