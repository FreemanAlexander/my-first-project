# My First AgentOS Project

Учебный проект для прохождения интенсива AgentOS: учусь работать с Claude Code, терминалом, Git, GitHub, permissions, skills и структурой AI-агента.

## Identity
- Пользователь — новичок в Claude Code, Cursor, терминале, Git и серверной работе.
- Объясняй простым русским языком: что делаем, зачем и какой следующий шаг.
- Не выдавай большие инструкции сразу: двигайся маленькими шагами и жди подтверждения.
- Уровень автономии: действуй сам только в обратимых операциях. Для удаления, push, миграций, sudo, новых зависимостей и изменения секретов — сначала спроси.

## Project
- Мы создаём продающий лендинг и Telegram-бота с админскими функциями в рамках интенсива AgentOS.
- Цель проекта — научиться вайбкодить с Claude Code: ставить задачи, править файлы, работать с GitHub, permissions, skills и базовым проектным контекстом.
- Это отдельный учебный проект в `/home/agentos/my-first-project`.
- Это не боевой Jarvis/Richard и не часть `/home/edgelab`.
- Главные проектные правила теперь лежат в `.claude/CLAUDE.md`; корневой `CLAUDE.md` оставляем синхронизированным, чтобы Claude Code точно видел контекст проекта.

### Principles
1. Plan Mode — для задач из 3+ шагов сначала план
2. Verification Before Done — никогда «готово» без доказательства
3. Demand Elegance — нетривиальный код → «есть ли элегантнее?»
4. Autonomous Bug Fixing — баг → читай логи → чини

### Plan
- План задачи → tasks/todo.md с чек-боксами
- Сверь со мной до начала
- Отмечай выполненное по ходу

### Subagents
- Используй субагентов для исследования, архитектуры, ревью и параллельной обработки больших задач.
- Для маленьких правок не используй субагентов — быстрее сделать напрямую.
- Максимум одновременно — 5.
- 1 субагент = 1 задача.

### Lessons
- Корректировка → запись в tasks/lessons.md
- Формат: что не так / как правильно / правило на будущее

## Boundaries

### Autonomy
GREEN (без вопросов): рефакторинг, тесты, докс, форматирование
RED (с моим OK): миграции БД, production deploy, удаление, новые зависимости, CI/CD

### Security
- НИКОГДА не коммить .env, .env.*, *.key, *.pem, secrets/
- Никогда не выводить токены, пароли и API-ключи в чат, stdout, логи или скриншоты.
- rm -rf, DROP TABLE, sudo — только с явным OK.
- Новые зависимости — сначала обоснуй зачем.

### Git
- Коммиты — на русском: «Добавил ...», «Исправил ...», «Обновил ...».
- Ветки: feature/, fix/, refactor/, docs/.
- НИКОГДА: push --force, удаление веток, push в main/master напрямую без явного OK.
- Перед коммитом показывай git status и кратко объясняй, что попадёт в коммит

## Style

### Язык
- Общение со мной — русский.
- Код, имена переменных, классов, функций и файлов — английский.
- Комментарии в коде — английский, если комментарий реально нужен.
- Коммиты — русский.

### Типы
- TypeScript: strict: true, никогда any (unknown + guard)
- Python: type hints на ВСЕ функции

### Anti-patterns
- console.log в production
- var в JS (только const/let)
- TODO без номера задачи
- Магические числа
- Функция > 60 строк → разбить
- Файл > 400 строк → разбить
