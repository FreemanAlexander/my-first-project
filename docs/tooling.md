# Tooling — линтеры, форматтеры, type-check

Этот документ описывает, какие инструменты планируется подключить в проекте и в каком порядке. Сейчас зафиксированы только конфиги, которые работают без установки зависимостей. Всё, что требует пакеты, отложено и подключается на соответствующем этапе из `tasks/todo.md`.

## Что уже добавлено (без зависимостей)

- **`.editorconfig`** — единые правила оформления файлов (LF, UTF-8, 2 пробела, 4 для Python, табы в Makefile). Подхватывается VS Code, Cursor, JetBrains и большинством редакторов из коробки.
- **`.gitattributes`** — нормализация переводов строк (`* text=auto eol=lf`), бинарные расширения, `linguist-generated` для lock-файлов.

## Отложенные решения

Эти инструменты требуют установки зависимостей, поэтому конфиги добавляем не сейчас, а на этапе, когда мы реально ставим стек (этап 2 — frontend, этап 3 — bot, этап 6 — tests/checks).

### Frontend (`web/`, Next.js 15 + TypeScript)

- **ESLint** — линт. Базовый конфиг `eslint-config-next` приходит вместе с `create-next-app`, поэтому отдельный `.eslintrc` пока не добавляем. Решение: использовать дефолтный `eslint-config-next` + `@typescript-eslint`. Подключаем на этапе 2 (Frontend) при инициализации Next.js.
- **Prettier** — форматирование. Конфиг (`.prettierrc`) добавим вместе с установкой `prettier` и `eslint-config-prettier`. Параметры: `singleQuote: true`, `trailingComma: "all"`, `printWidth: 100`. Подключаем на этапе 2.
- **TypeScript strict** — включается через `tsconfig.json`, который сгенерирует Next.js. Дополнительно потребуем `strict: true`, `noUncheckedIndexedAccess: true`. Подключаем на этапе 2.

### Bot / backend (`bot/`, Python 3.12 + aiogram)

- **Ruff** — линт + форматтер для Python (заменяет flake8 + isort + частично black). Конфиг — секция `[tool.ruff]` в `pyproject.toml`. Подключаем на этапе 3 (Bot) при инициализации Python-проекта.
- **mypy** — type-check для Python. Конфиг — секция `[tool.mypy]` в `pyproject.toml`, `strict = true`. Подключаем на этапе 3 или 6.
- **Black** — пока не берём: форматирование закрывается Ruff (`ruff format`). Если позже окажется, что Ruff недостаточно, вернёмся к этому решению.

### Общие проверки (этап 6 — Tests / checks)

- **pre-commit** — хук, который гоняет линт/формат/type-check перед коммитом и блокирует случайный коммит `.env`. Конфиг `.pre-commit-config.yaml` добавим на этапе 6.
- **GitHub Actions** — CI с шагами `lint` + `typecheck` + `test`. Файлы `.github/workflows/*.yml` добавим на этапе 6.

## Правило

Каждый раз, когда подключаем новый инструмент:

1. Сначала обосновываем зачем (правило проекта про новые зависимости).
2. Добавляем зависимость в нужный манифест (`package.json` / `pyproject.toml`).
3. Добавляем конфиг в репозиторий.
4. Обновляем этот документ: переносим пункт из «Отложенные решения» в «Что уже добавлено».
