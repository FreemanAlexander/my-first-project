## Главная идея

Marketer не должен ограничиваться сценарием:

`канал -> топ-10 видео`

Top-N используется только как fallback, если пользователь не задал критерий.

Основной режим:

`канал / автор / плейлист / список видео -> корпус материалов -> анализ по критерию пользователя`

## Что должен уметь Marketer

Если пользователь даёт одно видео:

- получить metadata;
- получить transcript при необходимости;
- вытащить хуки, структуру, CTA, идеи, цитаты, тайминги.

Если пользователь даёт канал или автора:

- собрать доступную карту видео;
- использовать названия, даты, просмотры, описания, комментарии и transcript выбранных видео;
- анализировать по критерию пользователя.

## Примеры критериев

- хуки;
- темы;
- повторяющиеся боли;
- комментарии / отзывы / возражения;
- сильные открытия;
- структура продажи;
- контентные паттерны;
- темы для PML;
- идеи для Reels / Shorts / Telegram;
- моменты для нарезок;
- видео, близкие к заданной теме.

## Defaults

Если критерий не задан:

- просмотреть recent/strong videos;
- найти повторяющиеся темы;
- выделить сильные заголовки и хуки;
- предложить идеи для PML.

## Safety

Ничего не публиковать, не комментировать, не лайкать и не менять без явной команды.
